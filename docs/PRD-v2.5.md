# 情绪记录 App 产品需求文档（PRD）

## 📋 文档信息

- **项目名称：** MoodTracker（暂定）
- **文档版本：** v2.5
- **创建日期：** 2026-03-22
- **更新日期：** 2026-03-28
- **产品负责人：** 朋朋
- **目标平台：** 移动端（iOS + Android）

## 🎯 产品愿景与目标

### Vision Statement
一个简洁、隐私优先的个人情绪记录与分析应用，帮助用户追踪情绪变化，发现情绪模式，提升自我认知。

### Core Value Prop
- 快速记录：3秒完成一次情绪记录
- 数据可控：完全本地存储，用户拥有数据所有权
- 渐进深入：从简单统计开始，逐步支持深度分析

### Jobs to be Done (JTBD)

| Job | Motivation | Outcome |
|-----|-----------|---------|
| 快速记录情绪状态 | 忙碌时不想浪费时间 | 3秒内完成记录 |
| 查看情绪趋势 | 想了解情绪变化模式 | 一目了然的图表 |
| 保持数据隐私 | 担心云端数据泄露 | 完全本地存储 |
| 导出数据备份 | 防止数据丢失 | 一键导出CSV |

## 🎯 产品目标与指标

### North Star Metric
周活跃用户留存率 (WAU Retention)
- 目标：≥ 40%（4周）
- 定义：第1周使用的用户，在第4周仍在使用

### Supporting Metrics

| 指标 | 目标 | 类型 |
|------|------|------|
| 日均记录次数 | ≥ 2次/天 | Leading |
| 平均完成记录时间 | ≤ 3秒 | Lagging |
| 用户满意度 (NPS) | ≥ 50 | Leading |
| 功能完成率 | ≥ 80%（MVP功能） | Lagging |
| 应用崩溃率 | < 0.1% | Lagging |

Leading Indicators (早期信号)
- 每日记录完成率
- 情绪类型分布（避免单一情绪）
- 功能使用频率（统计页 vs 记录页）

## 👥 目标用户画像

### Primary User: "自我觉察者"

- 年龄：18-35岁
- 职业：职场新人、学生、自由职业者
- 特征：
  - 关注心理健康和自我成长
  - 有反思习惯，但缺乏工具
  - 重视隐私，不喜欢数据被收集
  - 时间宝贵，想要快速记录
- 痛点：
  - 现有app功能过载、操作繁琐
  - 云端隐私担忧
  - 没有持续使用的动力
- 使用场景：
  - 工作间隙快速记录情绪
  - 睡前回顾一天的情绪
  - 情绪低落时寻找模式

### Secondary User: "心理爱好者"

- 特征：对情绪分析感兴趣，愿意花时间深入
- 需求：更高级的分析功能、可视化
- 使用场景：
  - 周末回顾情绪模式
  - 探索情绪触发因素

## 🚀 功能需求

按时间线组织 (Now/Next/Later)

### Now (Committed) - MVP (4-6周)

#### US-0: 首次引导流程（Onboarding）

作为一名新用户
我想要在首次打开App时获得引导
以便快速了解核心功能并开始使用

验收标准：
- ✓ 3页滑动引导卡片（记录情绪→查看趋势→隐私承诺）
- ✓ 底部"开始使用"按钮
- ✓ 右上角提供"跳过"选项
- ✓ 引导仅首次打开时显示，不可重复触发（除非重装）
- ✓ "我的"页面提供"使用引导"入口，允许用户主动重播
- ✓ 每页含插图/图标 + 一句核心文案
- ✓ 引导页使用浅色背景，过渡动画流畅（≤300ms/页）

📝 v2.5 更新：引导页新增重播入口，允许用户通过"我的"页主动回顾引导内容。

#### US-1: 情绪记录

作为一名自我觉察者
我想要快速记录当前的情绪状态
以便日后回顾和发现模式

验收标准：
- ✓ 可以从5种基础情绪中选择（开心、平静、难过、生气、焦虑）
- ✓ 情绪图标按效价（积极→消极）从左到右排列：开心→平静→难过→焦虑→生气
- ✓ 可以设置情绪强度（1-5级离散滑动条）
- ✓ 强度默认值为3（中间值），允许不调整直接保存
- ✓ 快速路径：选情绪→保存（2步完成，无需强制调整强度）
- ✓ 可选输入文字备注
- ✓ 点击保存后<500ms完成
- ✓ 保存成功时，选中情绪图标执行放大+粒子扩散动画（200ms内完成）
- ✓ 时间戳自动记录，点击时间文字弹出系统DateTime Picker，可选范围：当前时间往前7天内
- ✓ 备注输入时键盘弹出，页面整体上推，保存按钮固定在键盘上方（sticky bottom）
- ✓ 键盘工具栏增加"完成"按钮收起键盘
- ✓ 粒子动画低端机降级：运行时帧率检测，首帧渲染超过16ms（60fps基准）即自动降级为简单缩放

📝 v2.5 更新：
- 时间戳修改交互完善：点击时间文字弹出DateTime Picker，限制7天内
- 键盘弹出时UI适配：页面整体上推、保存按钮sticky bottom
- 粒子动画低端机降级检测标准：运行时帧率检测方案

#### US-2: 记录列表查看

作为一名自我觉察者
我想要查看我的历史情绪记录
以便回顾和修改

验收标准：
- ✓ 按时间倒序展示所有记录
- ✓ 每条记录显示：时间、情绪图标、强度条、备注预览
- ✓ 左滑删除记录（有确认提示）
- ✓ 右滑编辑记录
- ✓ 编辑页复用情绪记录表单（US-1），预填已有数据，保存后返回列表并显示Toast"已更新"
- ✓ 首次进入列表页时，显示一次引导动画：先右滑露出编辑→滑回→再左滑露出删除→滑回。底部显示文字提示"左滑可删除，右滑可编辑"
- ✓ 每张卡片右上角提供 ⋯ 更多菜单（备选入口），包含编辑和删除选项
- ✓ 下拉刷新列表
- ✓ 手势冲突处理：纵向优先（先判断是否下拉刷新，再判断左右滑动），滑动角度阈值 ±30°

📝 v2.5 更新：

- 滑动引导改为两步演示：右滑+左滑，底部增加文字提示

#### US-3: 情绪趋势统计

作为一名自我觉察者
我想要查看情绪趋势图表
以便了解情绪变化模式

验收标准：
- ✓ 支持7天/30天/全部时间范围切换
- ✓ 采用纵向滚动布局，按优先级排列：
  1. 统计摘要（平均强度、最频繁情绪、记录总数、记录天数）—— 最先可见
  2. 情绪分布（饼图）—— 其次展示
  3. 强度趋势（折线图）—— 深度用户细看
- ✓ 统计摘要中记录总数与记录天数搭配显示（如"47条 / 28天→平均每天1.7条"）
- ✓ 数据不足时（<7天记录）折叠图表区域，显示提示文案："记录更多天后解锁趋势分析"
- ✓ 图表加载时间<1秒
